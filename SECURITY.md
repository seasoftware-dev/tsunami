# Tsunami Browser Security Policy

The Tsunami Browser team takes security seriously. We appreciate responsible disclosure of security vulnerabilities and are committed to addressing issues promptly to keep users safe.

## Supported Versions

The following versions of Tsunami Browser currently receive security updates:

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Older releases | No |

If you are using an unsupported version, please report the vulnerability and then upgrade to the latest version.

## Reporting a Vulnerability

If you discover a security vulnerability in Tsunami Browser, **please do not disclose it publicly**.

Instead, report it privately using one of the following methods:

- Open a **private security advisory** on GitHub (preferred)
- Email: ryan@ryanpc.org or wyind@wyind.dev

When reporting, include as much detail as possible:

- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact
- Screenshots or logs if applicable

## Response Timeline

You can expect the following response timeline:

- **Acknowledgement**: Within 72 hours
- **Initial assessment**: Within 7 days
- **Fix or mitigation**: As soon as reasonably possible, depending on severity

Critical vulnerabilities will be prioritized.

## Disclosure Policy

Once a vulnerability has been resolved, coordinated disclosure may occur. Please allow time for a fix to be released before sharing details publicly.

## Scope

This security policy applies to:

- Tsunami Browser core application
- Official releases and builds
- First-party extensions or components included with Tsunami Browser

Third-party websites, extensions, or dependencies (Qt, Chromium, SQLite) are subject to their own security policies.

## Privacy Commitment

Tsunami Browser is committed to user privacy:

- No telemetry or data collection by default
- No built-in analytics or crash reporting
- All settings stored locally
- No connection to external servers except as initiated by the user
